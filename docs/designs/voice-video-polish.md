## Vision

### 10x Check
The 10x version of this plan would make OwnCord's voice/video not just match Discord but exceed it in areas Discord is weak: lower latency for small groups (no CDN overhead), per-user volume controls that persist across sessions, spatial audio in voice channels, and screen annotation tools during screenshare. The research phase would expand to include Zoom, Google Meet, and FaceTime for video-specific behaviors.

For this review, the accepted scope is "match Discord baseline + refactor the codebase to support future expansion." That's the right scope for now.

## Implementation Order

1. Research phase (human: ~2h hands-on testing). Output: comparison matrix saved to `docs/brain/06-Specs/VOICE-COMPARISON-MATRIX.md` covering 25+ behaviors across Discord, TeamSpeak, and optionally Guilded.
2. Generate DESIGN.md via /design-consultation (~15 min CC)
3. Refactor livekitSession.ts (1,509 lines) into 4 modules using facade pattern (~60-90 min CC). Existing exports stay in livekitSession.ts, delegating to new internal modules. No import site changes needed.
4. Migrate VAD to AudioWorklet (~30 min CC). Note: rnnoise-worklet.js/ts already exists for noise suppression. The VAD worklet is separate -- it handles voice gating (mute/unmute based on volume threshold), not noise reduction. Both worklets will run in the audio pipeline.
5. Write audio pipeline + event handler tests (~15 min CC)
6. Fix bugs: ghost state, 24h token TTL, concurrent join race (~20 min CC)
7. Add voice session metrics to /metrics (~5 min CC)
8. Implement UX fixes (each is a separate commit, ~2.5h CC total):
   a. Speaker indicators: green ring on avatar in voice channel list + sidebar (~20 min, CSS + voiceStore.speaking binding)
   b. Permission recovery: "Grant Microphone" button in VoiceWidget when in listen-only mode (~15 min, new UI element)
   c. Device hot-swap: listen to `navigator.mediaDevices.devicechange`, auto-fallback to default, toast notification (~20 min, deviceManager.ts)
   d. Screenshare volume slider: per-tile range input in VideoGrid using LiveKit's `setVolume()` (~15 min, extends existing mute control)
   e. Camera/screenshare stop feedback: toast notifications on toggle (~5 min, trivial)
   f. Retry UI: "Reconnect" button in VoiceWidget on connection failure (~15 min, new UI element)
   g. VAD indicator: small mic-level bar in VoiceWidget during call (~15 min, bind to existing analyser data)
   h. Connection quality warning: auto-expanding stats pane + yellow/red toast when quality degrades (~20 min, extends connectionStats.ts)
   i. Bandwidth display: human-readable Mbps in expanded stats pane (~10 min, extends connectionStats.ts)
9. Write HTTPS proxy unit tests (~15 min CC)
10. Build voice E2E test infrastructure (~30 min CC). Note: produces local-only test infra initially. LiveKit binary runs via Docker or `--dev` mode. CI integration is a follow-up task (requires Docker-in-CI setup).
11. Final: run all tests, manual QA against comparison matrix

Total CC time: ~6-7 hours (adjusted for refactor complexity)
Total human time (research + QA): ~3-4 hours
